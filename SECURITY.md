# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Testnet | Yes |
| Mainnet | Yes (when live) |

---

## Reporting a Vulnerability

SILVR takes security extremely seriously.

If you discover a security vulnerability please
DO NOT open a public GitHub Issue.

Instead report it privately to:

Twitter DM : @SILVRprotocol

Please include:
- Description of the vulnerability
- Steps to reproduce it
- Potential impact
- Your suggested fix if you have one

---

## Response Time

We will acknowledge your report within 48 hours.
We will provide a full response within 7 days.
We will credit you in the fix if you wish.

---

## Scope

The following are in scope for security reports:

- Consensus layer vulnerabilities
- Wallet key generation weaknesses
- Network protocol attacks
- Double spend possibilities
- DAO governance manipulation
- Treasury fund vulnerabilities
- Any issue that could cause loss of funds

---

## Out of Scope

- Issues in third party dependencies
- Social engineering attacks
- Physical access attacks

---

## Disclosure Policy

We follow responsible disclosure.
We ask for 90 days to fix before public disclosure.
We will never take legal action against good faith
security researchers.

---

## Recognition

Security researchers who responsibly disclose
vulnerabilities will be:

- Credited in the fix commit
- Mentioned in release notes
- Rewarded with SILVR from the community treasury
  if the issue is critical
